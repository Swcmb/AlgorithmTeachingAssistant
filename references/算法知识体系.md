# 算法知识体系

> **算法竞赛不是死记硬背，而是理解算法"为什么这样设计"**

## 按竞赛阶段推荐学习顺序

### 第一阶段：入门（1-2个月）

```
语言基础 ──→ 模拟/枚举 ──→ 基础排序
                              │
                              ↓
              前缀和/差分 ←── 二分查找
```

看 OI Wiki 页面:
- [算法基础简介](https://oi-wiki.org/basic/)
- [枚举](https://oi-wiki.org/basic/enumerate/)
- [排序简介](https://oi-wiki.org/basic/sort-intro/)
- [二分](https://oi-wiki.org/basic/binary/)

### 第二阶段：提高（2-4个月）

```
贪心 ──→ 基础DP ──→ 栈/队列应用
  │         │            │
  │         ↓            ↓
  │    记忆化搜索    BFS/DFS基础
  │         │            │
  └─────┬──┘            │
        ↓                ↓
     递归分治 ──────→ 拓扑排序
                       │
                       ↓
                  基础图论 + 并查集
```

看 OI Wiki 页面:
- [贪心](https://oi-wiki.org/basic/greedy/)
- [动态规划基础](https://oi-wiki.org/dp/basic/)
- [图论部分简介](https://oi-wiki.org/graph/)

### 第三阶段：进阶（4-8个月）

```
背包DP ──→ 区间DP ──→ 最短路 (Dijkstra/Floyd)
  │                      │
  ↓                      ↓
线段树 ←── 树状数组 ←── MST (Kruskal/Prim)
  │                      │
  ↓                      ↓
字符串哈希 ──→ KMP ──→ 二分图匹配
```

### 第四阶段：高级（8个月+）

```
状压DP ──→ 树形DP ──→ 网络流
  │                     │
  ↓                     ↓
数论基础 ──→ 后缀数组 ──→ 平衡树
  │                     │
  ↓                     ↓
LCA+树剖 ──→ 斜率优化 ──→ LCT/CDQ分治
```

## 常见题型与算法对应

| 题目特征 | 可能的算法/数据结构 |
|---------|-----------------|
| "n <= 20" | 状压DP、暴力枚举子集 |
| "n <= 100, 区间操作" | 区间DP |
| "n <= 2000" | O(n²) DP |
| "n <= 10^5, 区间查询修改" | 线段树/树状数组/分块 |
| "n <= 10^5, 多次询问区间和" | 前缀和 |
| "图最短路" | Dijkstra / SPFA(判断负环用Bellman-Ford) |
| "字符串匹配" | KMP / 字符串哈希 |
| "公共祖先" | LCA (倍增/Tarjan/树剖) |
| "求第k大" | 堆 / 快速选择 / 权值线段树 |
| "连通性问题" | 并查集 / Tarjan SCC |
| "最优方案" | 贪心 / DP |
| "博弈" | SG函数 / 对称策略 |
| "有向图无环" | 拓扑排序 / DAG DP |
| "多个字符串" | Trie / AC自动机 |
| "模运算/质数" | 数论 / 快速幂 / 逆元 |
| "离线查询" | 莫队 / CDQ分治 / 离线排序处理 |

## 每个模块学习检查清单

当学习某个算法模块时，用以下清单确保掌握：

```
□ 能用自己的话解释这个算法的核心思想
□ 能手写算法模板（不参考资料）
□ 知道时间复杂度，能解释为什么是这个复杂度
□ 做过至少 3 道该算法的题目
□ 能识别题目中的变式（题目不直接告诉你要用什么算法）
□ 知道常见的坑和边界情况
□ 知道这个算法和其他类似算法的区别（如Dijkstra vs SPFA）
```
